The Code

Complete JavaScript to fetch 5 users from a real public API

script.js 
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
//  Fetch 5 Users from JSONPlaceholder API
//  API URL: https://jsonplaceholder.typicode.com/users
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

// STEP 1 — Define the API endpoint URL
const API_URL = 'https://jsonplaceholder.typicode.com/users';

// STEP 2 — Create an async function
async function fetchFiveUsers() {

  try {

    // STEP 3 — Send a GET request to the API
    const response = await fetch(API_URL);

    // STEP 4 — Check if the request was successful
    if (!response.ok) {
      throw new Error(`Request failed with status: ${response.status}`);
    }

    // STEP 5 — Convert the raw response to JSON
    const allUsers = await response.json();

    // STEP 6 — Take only the first 5 users
    const fiveUsers = allUsers.slice(0, 5);

    // STEP 7 — Loop through and display each user
    fiveUsers.forEach((user, index) => {
      console.log(`\n👤 User ${index + 1}:`);
      console.log('   Name    :', user.name);
      console.log('   Username:', user.username);
      console.log('   Email   :', user.email);
      console.log('   City    :', user.address.city);
      console.log('   Company :', user.company.name);
    });

    // STEP 8 — Return the data in case you need it elsewhere
    return fiveUsers;

  } catch (error) {

    // STEP 9 — Handle any errors (network or server)
    console.error('❌ Something went wrong:', error.message);

  }
}

// STEP 10 — Call the function to run it
fetchFiveUsers();

Line-by-Line Interpretation

What every step means and why it's there

🔍 Code Breakdown — 10 Steps Explained
01
Define the API URL
const API_URL = 'https://...'
We store the URL in a constant variable at the top. This is good practice — if the URL ever changes, you only need to update it in one place, not scattered throughout the code. This API (JSONPlaceholder) is a free, real REST API used for testing.
02
Create an async function
async function fetchFiveUsers() { }
The async keyword marks this function as asynchronous. This means it can "pause and wait" for things like API responses without freezing the entire browser. It always returns a Promise behind the scenes.
03
Send the GET request with fetch()
const response = await fetch(API_URL)
fetch() sends an HTTP GET request to the server. The await keyword pauses the function here until the server responds. The result (response) contains the server's reply — but the actual data hasn't been read yet. Think of it like receiving a letter: you have the envelope, not the content inside yet.
04
Check if the request succeeded
if (!response.ok) throw new Error(...)
Important! fetch() does NOT automatically throw an error for status codes like 404 Not Found or 500 Server Error. response.ok is true only when the status is 200–299. If it fails, we manually throw an error to jump to the catch block.
05
Convert response to JSON with .json()
const allUsers = await response.json()
The server sends data as raw text. .json() reads and converts that text into a real JavaScript array/object that you can work with. We await it because this reading process also takes a small amount of time. After this line, allUsers is a JavaScript array of 10 user objects.
06
Take only the first 5 users with .slice()
const fiveUsers = allUsers.slice(0, 5)
The API returns 10 users total. .slice(0, 5) cuts the array from index 0 to index 4 (5 items), giving us only the first 5. The original allUsers array is not changed — slice creates a new array.
07
Loop through users with .forEach()
fiveUsers.forEach((user, index) => { ... })
.forEach() runs a function once for each item in the array. Each time it runs, user is the current user object and index is its position (0, 1, 2…). Inside, we read properties like user.name, user.email, and user.address.city (nested inside the address object).
08
Return the data
return fiveUsers
Returning the data makes the function reusable — other parts of your code can call await fetchFiveUsers() and get the 5 users back to do more things with them (like displaying them on a web page, as we do in the Live Demo below).
09
Handle errors with catch
catch (error) { console.error(...) }
If anything goes wrong inside the try block — a network failure, invalid URL, or the error we threw in Step 4 — JavaScript jumps here automatically. The error.message tells you exactly what happened. Never skip error handling in real apps.
10
Call the function
fetchFiveUsers()
Defining a function does nothing on its own — it's just a blueprint. You must call it to actually run the code. This line triggers everything: the fetch, the parsing, the loop. Since it's an async function, it runs in the background without blocking your page.

What Does the API Return?

The shape of each user object from the API

API Response — One User Object 
{
  id:       1,                          // unique ID number
  name:     "Leanne Graham",            // full name
  username: "Bret",                     // username → user.username
  email:    "Sincere@april.biz",       // email → user.email
  phone:    "1-770-736-8031 x56442",   // phone number
  website:  "hildegard.org",           // website

  address: {                           // nested object
    street:  "Kulas Light",
    city:    "Gwenborough",             // → user.address.city
    zipcode: "92998-3874",
    geo: { lat: "-37.31", lng: "81.14" }
  },

  company: {                           // nested object
    name: "Romaguera-Crona",           // → user.company.name
    bs:   "harness real-time e-markets"
  }
}
💡 Accessing nested data: For top-level fields use user.name. For data inside an object use user.address.city or user.company.name — just chain the dots deeper.
