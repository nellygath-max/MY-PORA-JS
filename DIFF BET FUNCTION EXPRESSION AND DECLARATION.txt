CONTINUATION OF FUNCTION EXPRESSION

Function Expressions
A function expression is a function stored in a variable
The variable name can be used to call the function

Basic idea
The function has no name (usually)
The variable acts like the function name

Example:

const greet = function () {
  console.log("Hello!");
};

greet();

What is happening here?
function () { ... } is the function
It is stored in the variable greet
You call it using greet()

Simple way to remember
Function declaration: named function written directly
Function expression: function stored inside a variable

FUNCTION DECLARATION:

Hoisting in JavaScript is a behavior where variable and function 
declarations are moved to the top of their scope before the code runs (during the compilation phase).

JavaScript “reads” your code twice:

First pass: it registers declarations (hoists them)
Second pass: it runs the code line by line

Example with var
console.log(x); // undefined
var x = 5;

What actually happens internally:

var x; // hoisted (declared first)
console.log(x); // undefined
x = 5;


Example with let and const
console.log(y);
let y = 10;

❌ This gives an error:

Cannot access 'y' before initialization

let y = 10;

❌ This gives an error:

Cannot access 'y' before initialization

👉 Why?

let and const are hoisted too
but they are in a temporal dead zone (TDZ) until the line where they are defined
🔹 Function hoisting

Function hoisting
sayHi();

function sayHi() {
  console.log("Hello!");
}

✔ This works perfectly because:

function declarations are fully hoisted (name + body)

Internally (what JS sees first):

function sayHi() {
  console.log("Hello!");
}

Then it runs the rest:

sayHi();

Why this works

Because:

✔ Function declarations are fully hoisted
✔ That means JavaScript already knows sayHi exists BEFORE it reaches the function line

So this is valid even though it looks reversed.

HOW JavaScript TREATS CONST AND LET:

Step-by-step internally:
JS sees the variable:

const sayHi;

It is hoisted (created early)

But it is in the Temporal Dead Zone (TDZ)
❌ You cannot use it yet
Only when JS reaches this line:

sayHi = function () { ... }

  return "Hello " + Chinelo;

The correct way of (using a parameter)

If you want to pass "Chinelo" into the function, you must define a parameter:

function greet(name) {
  return "Hello " + name;
}

console.log(greet("Chinelo"));

✅ Output:
Hello Chinelo

ANOTHER EXAMPLE:

What type of function is this? 
 
function greet() {
  return "Hello " + Chinelo;
}

✅ This is called a function declaration
❌ It is NOT a function expression.

Why?
A function declaration starts with the function keyword and has a name.
A function expression is when a function is stored in a variable.


Function declaration example

function greet() {
  return "Hello";
}

Function expression example:

const greet = function () {
  return "Hello";
};


Parameter = the variable inside the function definition
Argument = the value you pass when calling the function

Easy rule to remember:
Function declaration → function name() {}
Function expression → const name = function() {}
